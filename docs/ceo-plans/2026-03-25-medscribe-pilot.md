## Vision

### 10x Check
The 10x version: Debrief becomes the default feedback capture workflow across all Canadian medical schools. Every preceptor-trainee interaction generates structured, competency-mapped assessment data automatically. Programs have real-time visibility into trainee progression. The assessment data that drives accreditation decisions is actual evidence, not fiction. Longitudinal analytics show growth trajectories, flag struggling trainees early, and identify preceptors who give the best teaching feedback.

## Key Decisions (from review)

- **Platform scope:** Own the feedback workflow only. One45 continues to own program management and accreditation reporting.
- **Implementation approach:** Platform-ready MVP (Approach B from design doc)
- **Multi-trainee recordings:** One recording per trainee. Resident starts a new session for each.
- **PHI scrubbing order:** Regex first (fast/cheap), then LLM pass (contextual). Defense-in-depth.
- **Consent model:** Build in parallel with development. Adapt consent flow if PD meeting outcome requires changes. Not a hard gate.
- **Form engine:** Multi-form JSON template engine from day one (boil the lake). Not hardcoded.
- **LLM confidence scoring:** Keep for pilot. Even uncalibrated, it directs resident attention.
- **Infrastructure:** Supabase-only (DB + Auth + Storage + Edge Functions, all ca-central-1). Single provider.
- **Pipeline:** Async processing. Resident uploads, can close app, checks back when ready.
- **Resident incentive:** Type-A self-advocacy. Residents want their record to reflect actual performance. Morally distressing to get good verbal feedback captured as generic written form.
