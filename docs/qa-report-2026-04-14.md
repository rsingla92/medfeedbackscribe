## Detailed Findings

### BUG-001 — Hydration Mismatch (High)

**URL:** `/auth`, `/onboarding` (any page with text inputs)
**Console output:**
```
A tree hydrated but some attributes of the server rendered HTML didn't match
the client properties... style={{caret-color:"transparent"}}
```
**Root cause:** A browser password manager extension (e.g. 1Password, Bitwarden) injects `caret-color: transparent` into input fields on the client before React hydrates. Next.js 16/React 19 with strict hydration detects this mismatch and logs a warning + shows the "1 Issue" dev overlay.

**Impact:** Warning only — the app continues to work. But it pollutes the console and the Next.js dev overlay showing "1 Issue" may alarm developers. Not fixable in app code without suppressing hydration warnings (not recommended). The fix is to add `suppressHydrationWarning` to the affected `<input>` elements.

**Note:** This is a dev-mode-only issue; the "1 Issue" overlay does not appear in production builds.

### BUG-002 — Math.random() in Render (Medium) — FIXED

**URL:** `/demo` (recording step, active recording)
**File:** `src/app/demo/page.tsx` line 278
**Before:**
```tsx
style={{ height: `${20 + Math.random() * 80}%`, animation: ... }}
```
**After (fixed):** Precomputed `WAVEFORM_HEIGHTS` constant using deterministic sine-wave pattern at module scope. No random values during render.

### BUG-004 — Home Page Layout Breaks at 1440px (Medium)

**URL:** `/` (unauthenticated state)
**Viewport:** 1440px
**Description:** The unauthenticated home page `<header>` has no max-width constraint. At 1440px:
- Header text hugs far left edge
- Record button and Sign In CTA are horizontally centered

This creates a visually disconnected layout. DESIGN.md specifies `Max content width: 960px`. The authenticated `HomeClient` component may have proper max-width handling; the unauthenticated branch in `page.tsx` does not.

**Fix (not applied — structural change):** Wrap the unauthenticated `<main>` content in a `max-w-[960px] mx-auto` container, or add `max-w-md mx-auto` consistent with the mobile-first single-column design.

### BUG-005 — Terms of Service Not Linked (Medium)

**URL:** `/auth`
**Text:** "By signing in you agree to our terms of service."
**Issue:** The words "terms of service" are plain text, not a hyperlink. No ToS page exists in the codebase. This is a legal/compliance concern for a medical app handling PHI data.

**Fix (not applied — content creation needed):** Create `/terms` page or link to external ToS document.
